## 📋 3. Memahami Prinsip C.A.L.M.S

Saya menerapkan lima prinsip dasar DevOps dalam tugas ini:
1.  **Culture (Budaya):** Belajar disiplin dalam mendokumentasikan setiap langkah pengerjaan.
2.  **Automation (Otomatisasi):** Menggunakan perintah terminal untuk mempercepat proses kirim data.
3.  **Lean (Ramping):** Mengatur folder tugas agar rapi dan hanya berisi file yang diperlukan saja.
4.  **Measurement (Pengukuran):** Mengecek riwayat pengerjaan melalui log Git.
5.  **Sharing (Berbagi):** Mengunggah hasil kerja ke GitHub agar transparan dan bisa diperiksa.

## 🧠 4. Analisis Pengalaman & Refleksi Teknis

Proses pengerjaan praktikum pertemuan pertama ini memberikan tantangan yang cukup signifikan secara teknis maupun mental. Selama seharian melakukan konfigurasi, saya menghadapi beberapa kendala kritis dalam sinkronisasi repositori dan manajemen *state* pada Git.

**Detail Kendala yang Dihadapi:**
* **Kompleksitas Sinkronisasi:** Saya sempat mengalami hambatan saat mencoba menyelaraskan data lokal dengan repositori *remote*. Hal ini memaksa saya untuk melakukan penelusuran ulang (*troubleshooting*) terhadap riwayat *commit* dan memahami bagaimana Git mengelola objek file secara internal.
* **Integritas Data:** Terdapat momen di mana konfigurasi yang tidak sinkron menyebabkan hilangnya beberapa progres pengerjaan. Kondisi ini menuntut ketelitian tinggi untuk melakukan pemulihan data manual dan memastikan tidak ada informasi yang tumpang tindih dengan metadata lain.

**Hasil Analisis Mandiri:**
Kejadian tersebut memberikan pelajaran berharga bahwa DevOps bukan sekadar tentang keberhasilan menjalankan perintah, melainkan tentang **resiliensi** (ketahanan) dalam menghadapi anomali sistem. Saya belajar bahwa dokumentasi yang rapi dan pemahaman terhadap perintah *force-update* adalah kunci dalam menjaga integritas sebuah proyek ketika alur kerja standar mengalami hambatan.

**Kesimpulan:**
Meskipun proses pengerjaan memakan waktu yang jauh lebih lama dari yang diperkirakan akibat kendala teknis tersebut, hal ini justru memperdalam pemahaman saya mengenai kontrol versi di level yang lebih praktis. Saya kini jauh lebih siap dalam mengantisipasi konflik data pada pertemuan praktikum selanjutnya.
